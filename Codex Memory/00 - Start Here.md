# Codex Memory

这是 Codex 的跨项目长期记忆入口。它保存经过确认、未来仍有价值的规则、偏好、工作流、决策和项目背景；它不是聊天记录归档，也不是密码库。

## 每次任务开始时

Codex 按以下顺序读取：

1. [[10 - Core/Rules|长期规则]]
2. [[10 - Core/Preferences|个人偏好]]
3. [[10 - Core/Workflows|常用工作流]]
4. [[10 - Core/Decisions|跨项目决策]]
5. 如果能识别当前项目，再读取 `20 - Projects` 中对应的项目记忆

如果本次任务与某类记忆无关，可以只读取标题和相关章节，避免把整个知识库塞入上下文。

## 写入原则

- 只写入用户明确确认，或在任务中有清晰证据且未来仍有价值的信息。
- 用户当前指令始终高于这里的旧记忆；发现冲突时先遵循当前指令，再更新或标记旧条目。
- 优先更新已有条目，不重复堆叠同一事实。
- 每条重要记忆包含日期、来源和状态；不确定内容标为 `待确认`。
- 不把一次性任务细节、未经证实的猜测、完整聊天记录写成长期记忆。
- 外部事实可能过期，使用前仍需核验；记忆只保存上下文，不代替事实来源。
- 任务结束时，仅在形成了新的长期偏好、规则、可复用流程或明确决策时更新本库。

## 安全边界

不要在本库保存密码、API 密钥、访问令牌、私钥、恢复码或其他认证秘密。此类信息应放入系统钥匙串或专用密码管理器；这里只能记录秘密的名称、用途和安全存放位置，不记录秘密本身。

## 目录

- `10 - Core`：所有项目共享的稳定记忆
- `20 - Projects`：按项目隔离的背景、约束与状态
- `90 - Inbox`：待用户确认后再归档的候选记忆
- `Templates`：新增记忆时使用的模板
